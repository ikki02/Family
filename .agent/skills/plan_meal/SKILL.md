# 献立を考える
Notion MCP経由で直近の「献立」を読み取って、おすすめの献立を提案してください。

## 手順
- Notion MCP経由で直近の「献立」を読み取る
  - 指定がなければ、直近2週間分を読み取る
- 直近の献立の主菜について、肉類、魚類、野菜、の傾向を振り返る
- もし使いたい食材の指定があれば、それを優先する。
  - 例: キャベツ、大根を使いたい、など。
- なるべく同じ傾向が続かないように、提案する献立のバランスを考える
- 1ヶ月以上前の献立は参照して返答に含めても良い
- 3週間以上前の⭐️がついたレシピは1つ含めても良い

## 取得方法のヒント
- **データベースID**: `81f4fd74-9413-8257-869c-811a2b46e3a2`
- **データソースID**: `8734fd74-9413-821b-adc1-87dfb37410ee`
- **ビューID**: `1604fd74-9413-8238-b6dc-88bafd007c0e` （直近1週間をフィルタリングするためのビュー）
- **重要な制約**:
    - **タイトル参照禁止**: 「名前（タイトル）」列はほとんどが空であるため、検索のクエリや判定の根拠にしないこと。
    - **類推・逆引き禁止**: 「買い物リスト」からの類推や、レシピ側からの逆引きアプローチは行わないこと。
- **重要なプロパティ**:
    - **日付** (`date:日付:start`): 献立の実行予定日。ここが最も重要な判定基準となる。
    - **レシピ** (`レシピ`): 実際の内容を確認するためにこのリレーションを辿る。
- **絞り込みと検索のコツ**:
    - **作成日フィルター**: `mcp_notion_notion-search` の `filters` で `created_date_range` を活用し、直近（1〜2週間程度）に作成・更新されたエントリを絞り込むと効率的。
    - **一括取得**: インデックスや関連性順位に頼らず、作成日フィルターでヒットしたエントリのプロパティを一つずつ確認して、目的の日付のエントリを特定する。

## 出力の例
2/9昼: 豚肉の生姜焼き  
2/9夜: 鮭の塩焼き  
2/10昼: 鶏肉の唐揚げ  
2/10夜: 牛丼
